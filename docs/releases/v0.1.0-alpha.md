# v0.1.0-alpha Release Notes Draft

Status: unreleased, unstable, and not ready to tag.

Do not publish or tag `v0.1.0-alpha` until
[`public-alpha-validation-record.md`](../public-alpha-validation-record.md)
has complete sanitized evidence for every required gate and reviewer sign-off
says the release is ready.

## Who Should Try This

This alpha is intended for operators and contributors who are comfortable
reviewing Ansible, generated Docker Compose, encrypted configuration, service
state paths, and host changes before applying them.

Good candidates:

- contributors validating the first public release gates;
- homelab operators testing on disposable or non-critical hosts;
- reviewers checking docs, role contracts, validation output, and secret
  handling.

## Who Should Wait

Wait for a later release if you need:

- production-ready operations;
- broad app catalog coverage;
- hands-off upgrades;
- managed support;
- guaranteed backup, restore, and migration behavior across real workloads.

## Validation Blocker

The public alpha is blocked until these records are complete:

- [Public alpha evidence checklist](../public-alpha-checklist.md)
- [Public alpha validation record](../public-alpha-validation-record.md)
- [Demo and sample output](../demo-and-sample-output.md)

Current validation status: incomplete. The release is not ready.

## Manual GitHub Polish

Before tagging, verify repository metadata:

- repository description matches prototype alpha status;
- topics include relevant public discovery terms such as `ansible`,
  `docker-compose`, `homelab`, `self-hosted`, `ai`, `mcp`, and `automation`;
- README and release notes both say the release is unstable and not
  production-ready;
- CI status is visible from the default branch.

Record the sanitized review result in the validation record before release.
