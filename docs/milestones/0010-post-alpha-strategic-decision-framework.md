# 0010: Post-Alpha Strategic Decision Framework

Milestone 10 defines how ChimerAI will choose its post-alpha direction after
public alpha evidence starts coming in.

## Goals

- Add a public framework for interpreting post-alpha feedback.
- Name the main decision paths without claiming any path has already won.
- Define evidence thresholds for foundation work, Authentik automation
  extraction, MCP wiring, and app roles.
- Clarify high-level beta and stable meanings without making release claims.
- Add guardrails against premature pivots, extraction claims, and app-bundle
  drift.

## Non-Goals

- No beta or stable release claim.
- No commitment to extract Authentik automation.
- No commitment to become an MCP-only gateway, registry, or app bundle.
- No domain-specific product direction.
- No implementation of new roles, Ansible behavior, or runtime services.

## Deliverables

- [Post-alpha strategic decision framework](../post-alpha-strategic-decision-framework.md).
- Roadmap entry in [milestones README](README.md).
- Comparison-guide note that ChimerAI direction is evidence-driven.
- Role-governance guardrail for app roles and extraction.

## Completion Criteria

- The framework names feedback inputs, decision paths, evidence thresholds,
  beta and stable meanings, and pivot guardrails.
- The comparison guide says ChimerAI is not currently an MCP-only gateway, app
  bundle, or extracted Authentik collection.
- Role governance requires contract maturity, validation, owner, lifecycle
  expectations, and real adoption evidence before app-role expansion or
  extraction.
- Public docs avoid stability claims and premature extraction language.
